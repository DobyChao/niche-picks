# 小众点评 - Spec 草稿

## 定位
私人/小圈子的探店日记，记录真实的探店体验。

## 数据模型（两层）

### 🏪 店铺 (Shop)
- 名称
- 地址
- 分类
- 电话
- 营业时间
- **Geo 坐标**（经纬度，便于地图定位和空间搜索）
- **特色标签/关键词**（便于搜索和推荐）

### 📝 点评 (Review)
- 关联店铺
- 作者（昵称）
- 评分
- 文字评价
- 标签（如"适合约会""别带小孩""要预约"）
- 人均消费
- 探店日期
- **支持增删改**（非 append-only）

## 数据流（Git-like）
- 本地优先，数据存 IndexedDB
- 用户主动同步上传（需要邀请 token）
- 管理员通过 **diff 视图** 审核 merge（类似 wiki 编辑历史）
- 用户看到的是 **本地 + remote 合并视图**

### 数据来源切换
- 展示时如果数据来自本地和远程，按条为单位展示
- 每条可标识来源（remote/local）
- 点击可切换查看本地版本或远程版本

## Token 机制
- **用户类 Token**：用于发起同步请求
- **管理类 Token**：用于审核合并操作

## 用户系统
- 本地输入昵称即可使用
- 同步时需提供邀请 token

## 布局（响应式）
- 📱 手机：搜索栏 → 地图 → 时间线，纵向堆叠
- 💻 PC：左 3/4 地图 + 右 1/4 搜索/列表/点评详情
- 三个维度：地图（空间）、时间线（时间）、搜索（意图）

## 技术栈
- **Next.js** 全栈
- **高德地图 AMap**
- **Dexie.js** 封装 IndexedDB（推荐，提供类 SQL/ORM 查询 + React Hook useLiveQuery）
- **Tailwind CSS** 响应式样式
